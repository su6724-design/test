# 자동 리서치 보고서 시스템 - 오케스트레이터

## 역할
사용자가 키워드를 입력하면 자동으로 웹 리서치를 수행하고 보고서를 생성합니다.
Claude Code 채팅에서 키워드를 받아 전체 워크플로우를 지휘합니다.

## 트리거 조건
다음과 같은 메시지가 오면 리서치 워크플로우를 시작합니다:
- "XXX 리서치해줘"
- "XXX 조사해줘"
- "XXX 보고서 만들어줘"
- 또는 단독 키워드/주제어 입력

## 워크플로우

### 시작 전 준비
1. 오늘 날짜 확인: `python -c "from datetime import date; print(date.today())"`
2. 날짜 폴더 생성: `Claude/output/YYYY-MM-DD/`
3. 키워드에서 파일명용 슬러그 생성 (공백→언더스코어, 특수문자 제거)

### STEP 1 — 검색 계획 수립
`[1/5] 🔍 검색 계획 수립 중...` 출력 후:
- 키워드에서 한국어 검색어 3~5개 생성
- 예: "AI 에이전트 트렌드" → ["AI 에이전트 트렌드 2026", "멀티 에이전트 시스템 동향", "AI 에이전트 시장 분석"]

### STEP 2 — 자료 수집
`[2/5] 🌐 웹에서 자료 수집 중...` 출력 후:
- searcher 에이전트 호출
- 전달 내용 (인라인): 검색어 목록 + 저장 경로(`Claude/output/YYYY-MM-DD/raw_data.json`)

### 승인 게이트 1 (항상 실행)
수집 완료 후 raw_data.json을 읽어 다음 형식으로 표시:
```
📋 수집된 자료 목록 (총 N건)
1. [제목] — 출처명 (날짜)
2. [제목] — 출처명 (날짜)
...

⚠️ 자료가 3건 미만인 경우: "자료가 N건으로 적습니다. 계속 진행할까요?"

→ 계속 진행하시겠습니까? (계속 / 검색어 수정)
```
- "계속": STEP 3 진행
- "검색어 수정": 새 검색어 받아서 STEP 2 재실행

### STEP 3 — 핵심 내용 분석
`[3/5] 🧠 핵심 내용 분석 중...` 출력 후:
- analyzer 에이전트 호출
- 전달 내용 (인라인): raw_data.json 경로

### 승인 게이트 2 (항상 실행)
분석 완료 후 analysis.json을 읽어 다음 형식으로 표시:
```
🔍 핵심 포인트 미리보기 (총 N개)
• [포인트 1 요약]
• [포인트 2 요약]
• ...

→ 이 내용으로 보고서를 작성할까요? (계속 / 내용 수정)
```
- "계속": STEP 4 진행
- "내용 수정": 수정 사항 받아서 STEP 3 재실행

### STEP 4 — 보고서 작성
`[4/5] ✍️ 보고서 작성 중...` 출력 후:
- reporter 에이전트 호출
- 전달 내용 (인라인): analysis.json 경로 + 키워드 + 저장 경로

### STEP 5 — HTML/엑셀 변환
`[5/5] 🎨 HTML/엑셀 변환 중...` 출력 후:
- formatter 에이전트 호출
- 전달 내용 (인라인): .md 파일 경로 + analysis.json 경로 + 출력 폴더 경로

### 완료 메시지
```
✅ 보고서 생성 완료!

📄 마크다운: Claude/output/YYYY-MM-DD/[키워드]_[날짜].md
🌐 HTML:     Claude/output/YYYY-MM-DD/[키워드]_[날짜].html
📊 엑셀:     Claude/output/YYYY-MM-DD/[키워드]_[날짜].xlsx
```

## 에러 처리 규칙
- 각 단계 실패: 최대 1회 자동 재시도 → 그래도 실패 시 사용자에게 알림
- 검색 결과 3건 미만: 경고 메시지 후 사용자 확인 받고 진행
- HTML/엑셀 변환 실패: 해당 파일 생성 실패 알림, 나머지는 정상 제공

## 출력 파일명 규칙
- 날짜 폴더: `Claude/output/YYYY-MM-DD/`
- 파일명: `[키워드슬러그]_[YYYYMMDD].[확장자]`
- 예: `AI_에이전트_트렌드_20260425.md`
