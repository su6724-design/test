## 1. 작업 컨텍스트

### 배경과 목적

> 키워드 하나를 Claude Code 채팅에 입력하면, AI가 인터넷을 검색하고 보고서를 자동으로 만들어주는 시스템

사람이 직접 하면 2~3시간 걸리는 리서치 작업을 **5분 이내**로 줄이는 것이 목표입니다.

### 범위

- 포함: 웹 검색 → 분석 → 보고서(.md) → HTML 변환 → 엑셀 정리
- 제외: 로그인이 필요한 사이트 수집, 이미지/동영상 분석
- 검색 언어: **한국어만** (국내 자료 위주)

### 입력 / 출력

| 구분 | 내용 | 예시 |
|------|------|------|
| **입력** | Claude Code 채팅에 키워드 입력 | `"AI 에이전트 트렌드"` |
| **출력 1** | 마크다운 보고서 (3~5페이지) | `output/2026-04-25/AI_에이전트_트렌드_20260425.md` |
| **출력 2** | HTML 보고서 (흰 배경, 깔끔) | `output/2026-04-25/AI_에이전트_트렌드_20260425.html` |
| **출력 3** | 엑셀 정리본 (2개 시트) | `output/2026-04-25/AI_에이전트_트렌드_20260425.xlsx` |
| **중간 데이터** | 수집 원본 (날짜별 보관) | `output/2026-04-25/raw_data.json` |
| **중간 데이터** | 분석 결과 (날짜별 보관) | `output/2026-04-25/analysis.json` |

### 제약 조건

- 웹 검색: Claude Code 내장 WebSearch/WebFetch 도구 사용 (외부 API 불필요)
- 검색 결과는 최신 정보 우선 (1년 이내)
- 출처가 불분명한 자료는 제외
- 보고서는 한국어로 작성 (출처 URL은 원문 유지)
- 파이썬 라이브러리: `markdown`, `openpyxl` (무료 오픈소스)

### 용어 정의

| 용어 | 뜻 (쉽게) |
|------|----------|
| **오케스트레이터** | 전체 팀을 지휘하는 팀장 AI (CLAUDE.md) |
| **서브에이전트** | 팀장 지시를 받아 특정 역할을 수행하는 팀원 AI |
| **스킬** | 여러 에이전트가 공유해서 쓰는 도구 설명서 (SKILL.md) |
| **승인 게이트** | "이대로 계속 진행할까요?" 라고 사람에게 확인하는 지점 |
| **raw_data.json** | 검색으로 수집한 원본 데이터를 담는 파일 |
| **analysis.json** | 원본에서 핵심만 추출한 분석 결과 파일 |

## 2. 워크플로우 정의

### 전체 흐름도

```
사용자: Claude Code 채팅에 키워드 입력
        │
        ▼
[STEP 1] 오케스트레이터 — 검색 계획 수립 + 날짜 폴더 생성
  "[1/5] 🔍 검색 계획 수립 중..."
        │
        ▼
[STEP 2] searcher 에이전트 — 한국어 웹 검색 + raw_data.json 저장
  "[2/5] 🌐 웹에서 자료 수집 중..."
        │
        ▼
★ 승인 게이트 1 ★ — 수집된 출처 목록 확인 (항상)
        │ 계속 진행      │ 검색어 수정
        │               └─ STEP 2 재실행
        ▼
[STEP 3] analyzer 에이전트 — 핵심 내용 추출 + analysis.json 저장
  "[3/5] 🧠 핵심 내용 분석 중..."
        │
        ▼
★ 승인 게이트 2 ★ — 핵심 포인트 미리보기 확인 (항상)
        │ 계속 진행      │ 내용 수정
        │               └─ STEP 3 재실행
        ▼
[STEP 4] reporter 에이전트 — .md 보고서 작성 (3~5페이지)
  "[4/5] ✍️ 보고서 작성 중..."
        │
        ▼
[STEP 5] formatter 에이전트 — HTML 변환 → 엑셀 변환 (순차)
  "[5/5] 🎨 HTML/엑셀 변환 중..."
        │
        ▼
완료: 생성된 파일 3개 경로 안내
```

---

### 단계별 상세 정의

#### STEP 1 — 검색 계획 수립

| 항목 | 내용 |
|------|------|
| **담당** | 오케스트레이터 (CLAUDE.md) |
| **처리 방식** | LLM 판단 — 한국어 검색어 3~5개 생성 |
| **부수 작업** | `output/YYYY-MM-DD/` 날짜 폴더 생성 |
| **성공 기준** | 검색어 3개 이상 생성 |
| **검증 방법** | 규칙 기반 (개수 확인) |
| **실패 시** | 자동 재시도 1회 → 실패 시 사용자 알림 |

---

#### STEP 2 — 웹 자료 수집

| 항목 | 내용 |
|------|------|
| **담당** | searcher 에이전트 |
| **처리 방식** | Claude Code 내장 WebSearch + WebFetch 사용 |
| **검색 언어** | 한국어만 |
| **성공 기준** | 3건 이상 (5건 미만 시 "자료가 N건입니다" 알림 후 진행) |
| **검증 방법** | 규칙 기반 (개수 확인) |
| **실패 시** | 검색어 바꿔 자동 재시도 최대 3회 → 실패 시 에스컬레이션 |
| **저장 위치** | `output/YYYY-MM-DD/raw_data.json` |

---

#### ★ 승인 게이트 1 — 수집 자료 확인 (항상 실행)

사용자에게 보여줄 내용:
```
📋 수집된 자료 목록 (총 N건)
1. [제목] — 출처명 (날짜)
2. [제목] — 출처명 (날짜)
...
→ 이 자료로 보고서를 작성할까요? [계속 진행 / 검색어 수정]
```

| 응답 | 처리 |
|------|------|
| 계속 진행 | STEP 3으로 이동 |
| 검색어 수정 | 사용자 입력 받아 STEP 2 재실행 |

---

#### STEP 3 — 핵심 내용 분석

| 항목 | 내용 |
|------|------|
| **담당** | analyzer 에이전트 |
| **처리 방식** | LLM 판단 — raw_data.json 읽고 핵심 포인트 추출 |
| **성공 기준** | 핵심 포인트 3개 이상, 중복 없음, 출처 링크 포함 |
| **검증 방법** | LLM 자기 검증 |
| **실패 시** | 자동 재시도 최대 2회 → 실패 시 에스컬레이션 |
| **저장 위치** | `output/YYYY-MM-DD/analysis.json` |

---

#### ★ 승인 게이트 2 — 분석 결과 확인 (항상 실행)

사용자에게 보여줄 내용:
```
🔍 핵심 포인트 미리보기 (총 N개)
• 포인트 1 요약
• 포인트 2 요약
...
→ 이 내용으로 보고서를 작성할까요? [계속 진행 / 내용 수정]
```

---

#### STEP 4 — 보고서 작성

| 항목 | 내용 |
|------|------|
| **담당** | reporter 에이전트 |
| **처리 방식** | LLM 생성 — analysis.json 읽고 .md 보고서 작성 |
| **분량** | **3~5페이지** |
| **성공 기준** | 필수 섹션 5개 모두 포함 |
| **검증 방법** | 규칙 기반 (섹션 제목 존재 여부) + LLM 자기 검증 |
| **실패 시** | 자동 재시도 1회 → 실패 시 에스컬레이션 |
| **저장 위치** | `output/YYYY-MM-DD/[키워드]_[날짜].md` |

---

#### STEP 5 — HTML + 엑셀 변환 (순차 처리)

| 항목 | 내용 |
|------|------|
| **담당** | formatter 에이전트 |
| **처리 방식** | 스크립트 실행 — HTML 먼저, 그 다음 엑셀 (순차) |
| **HTML 스타일** | 흰 배경, 깔끔한 보고서 스타일, 인쇄 가능 |
| **엑셀 시트** | 2개: ① 요약 시트 ② 출처 목록 시트 |
| **실패 시** | 하나 실패해도 다른 하나는 계속 진행 (스킵 + 로그) |
| **저장 위치** | `output/YYYY-MM-DD/[키워드]_[날짜].html` / `.xlsx` |

---

### LLM 판단 vs 코드 처리 구분표

| AI가 직접 판단 | 코드(스크립트)가 처리 |
|--------------|-------------------|
| 어떤 검색어가 좋을지 결정 | WebSearch/WebFetch 실행 |
| 어떤 내용이 핵심인지 판단 | JSON 파일 읽기/쓰기 |
| 보고서 글쓰기 | .md → .html 변환 |
| 출처 신뢰도 평가 | 엑셀 파일 생성 |
| 분석 품질 자기 점검 | 날짜 폴더 생성 |

## 3. 구현 스펙

### 폴더 구조

```
C:\Antigravity\
│
├── CLAUDE.md                              ← 오케스트레이터 지침 (팀장)
│
├── .claude/
│   ├── agents/
│   │   ├── searcher.md                    ← 검색 에이전트
│   │   ├── analyzer.md                    ← 분석 에이전트
│   │   ├── reporter.md                    ← 보고서 에이전트

│   │
│   └── skills/
│       ├── web-search/
│       │   └── SKILL.md                   ← 웹 검색 가이드 (SKILL.md 고정)

├── output/                                ← 결과물 저장 루트
│   └── YYYY-MM-DD/                        ← 날짜별 폴더 (자동 생성)
│       ├── raw_data.json                  ← 수집 원본
│       ├── analysis.json                  ← 분석 결과
│       ├── [키워드]_[날짜].md
│       ├── [키워드]_[날짜].html
│       └── [키워드]_[날짜].xlsx
│
├── templates/
│   └── report_style.css                   ← HTML 스타일 (참고용)
│
└── docs/
    └── AGENT-SYSTEM-PLAN.md               ← 이 파일
```

### CLAUDE.md 핵심 섹션 목록

| 섹션 | 내용 |
|------|------|
| 트리거 조건 | 사용자 메시지가 리서치 키워드인지 판단 기준 |
| 워크플로우 | STEP 1~5 순서 및 각 에이전트 호출 조건 |
| 진행 상태 표시 | `[N/5] 이모지 작업명...` 형식 |
| 승인 게이트 | 게이트 위치, 사용자에게 보여줄 포맷 |
| 에러 처리 | 재시도/에스컬레이션/스킵 규칙 |
| 출력 규칙 | 날짜 폴더 + 파일명 규칙 |

### 서브에이전트 상세 설계

> 파일 형식: YAML frontmatter(`---` 사이) + 마크다운 시스템 프롬프트
> `name`과 `description`은 필수.

#### searcher

```yaml
name: searcher
tools: WebSearch, WebFetch, Write, Bash
model: sonnet
skills: [web-search]   ← 시작 시 SKILL.md 전체 내용 주입
```

#### analyzer

```yaml
name: analyzer
tools: Read, Write
model: opus            ← 분석 품질이 전체 보고서에 영향
skills: 없음
```

#### reporter

```yaml
name: reporter
tools: Read, Write
model: sonnet
skills: 없음
```

#### formatter

```yaml
name: formatter
tools: Read, Write, Bash
model: haiku           ← 단순 변환 작업, 비용 절감
skills: [file-export]  ← 스크립트 경로 포함
```

---

### 스킬 목록

| 스킬 | 파일 경로 | 핵심 설정 |
|------|-----------|-----------|
| `web-search` | `.claude/skills/web-search/SKILL.md` | `disable-model-invocation: true` |
| `file-export` | `.claude/skills/file-export/SKILL.md` | `disable-model-invocation: true`, `allowed-tools: Bash(python *)` |

> 두 스킬 모두 `disable-model-invocation: true` → 오케스트레이터가 자동 호출 안 함.
> 각각 searcher, formatter 에이전트의 `skills:` 필드로만 로드됨.

---

### 데이터 전달 방식

| 구간 | 방식 | 이유 |
|------|------|------|
| 오케스트레이터 → searcher | 프롬프트 인라인 | 검색어 + 저장 경로 (짧음) |
| searcher → analyzer | 파일 기반 (`raw_data.json`) | 수집 데이터 크고 구조화됨 |
| analyzer → reporter | 파일 기반 (`analysis.json`) | 구조화된 JSON |
| reporter → formatter | 파일 경로만 인라인 | 경로는 한 줄 |

---

### 주요 산출물 파일 형식

#### raw_data.json
```json
{
  "keyword": "AI 에이전트 트렌드",
  "collected_at": "2026-04-25T10:00:00",
  "count": 7,
  "sources": [
    {"title": "...", "url": "...", "date": "...", "content": "..."}
  ]
}
```

#### analysis.json
```json
{
  "keyword": "AI 에이전트 트렌드",
  "analyzed_at": "2026-04-25T10:05:00",
  "summary": "3줄 요약",
  "key_points": [
    {"point": "...", "importance": "상/중/하", "source_url": "..."}
  ]
}
```

#### 엑셀 시트 구조
- **요약 시트**: 키워드 / 생성일 / 핵심 요약 / 출처 수
- **출처 목록 시트**: 번호 / 제목 / URL / 날짜 / 중요도

## 4. 인터뷰 결정사항

| 질문 | 결정 |
|------|------|
| 승인 게이트 방식 | 항상 2회 확인 (항상 확인) |
| formatter 처리 방식 | 순차 처리 (HTML → 엑셀) |
| 보고서 분량 | 표준형 (3~5페이지) |
| 검색 언어 | 한국어만 |
| 엑셀 시트 구성 | 요약 시트 + 출처 목록 시트 (2개) |
| HTML 디자인 | 깔끔한 흰 배경 (보고서 스타일) |
| 검색 실패 처리 | 3건 이상이면 알림 후 진행 |
| 실행 방법 | Claude Code 채팅에서 키워드 입력 |
| 중간 파일 보관 | 날짜별 폴더에 보관 (`output/YYYY-MM-DD/`) |
| 진행 상황 표시 | 단계별 상태 메시지 (`[N/5] 이모지 작업명`) |

---

> **구현 완료.** 실제 사용하려면 Claude Code 채팅창에 리서치할 키워드를 입력하세요.
> 예: `"AI 에이전트 트렌드 리서치해줘"`
